# M1 First-run Experience Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship milestone M1 (GitHub issues #41–#45): make the LinkedIn-provided identity visible in profile setup, defer the geolocation permission to first search, add tag suggestion chips, add a one-time three-step welcome intro, and commit the M1 funnel metrics doc.

**Architecture:** Pure-frontend batch on branch `feat/m1-first-run` — a Vite + React 19 + Tailwind SPA (no router; screens are conditional renders in `src/App.tsx`). Backend (InsForge) is NOT touched: no changes under `insforge/` or `migrations/`, no new env vars, no new client events. The DB already permits NULL coordinates (paired CHECK constraint) and `people-search` already 409s (`PROFILE_SETUP_REQUIRED`) on locationless callers, so Task 2 is frontend + docs only.

**Tech Stack:** React 19 function components + hooks, TypeScript strict, Tailwind (custom tokens), `@insforge/sdk` via existing `src/lib/api.ts`. Vitest is introduced in Task 3 for pure-logic tests only.

## Global Constraints

- **Design system:** follow `DESIGN.md` (repo root) and the idioms of neighboring components. Concretely: Tailwind custom tokens only (`ink`, `charcoal`, `slate`, `steel`, `stone`, `muted`, `cream`, `cream-light`, `cream-deeper`, `beige`, `canvas`, `surface`, `hairline`, `hairline-soft`, `hairline-strong`, `primary`, `primary-deep`, `success`, `signal`); display headings use `font-editorial`; cards are `rounded-xl border border-hairline-soft bg-canvas shadow-subtle` (hover `shadow-card`); pills are `rounded-full bg-cream-deeper px-2.5 py-1 text-xs font-medium`; primary buttons `rounded-lg bg-ink text-white hover:bg-charcoal`, secondary `border border-hairline-strong bg-canvas hover:border-ink`; interactive elements keep `focus-visible:outline focus-visible:outline-2 focus-visible:outline-offset-2 focus-visible:outline-primary` (or `-ink`); errors use `role="alert"` and `text-signal`. Never hardcode hex colors in components.
- **Animation:** CSS-only; every transition/animation must be gated with Tailwind `motion-safe:` variants (or a `prefers-reduced-motion` check where JS is unavoidable).
- **UI copy:** English, sentence case, matching the app's existing voice (e.g. "We could not…", privacy reassurance phrasing). Exact strings given per task are binding.
- **No backend changes:** nothing under `insforge/`, `migrations/`, `.github/`; no new environment variables; client may only send the three allowlisted events (`sign_in_completed`, `profile_completed`, `profile_updated`).
- **Verification per task:** `npm run typecheck && npm run lint && npm run build` must all pass before committing. Task 3 additionally runs `npm run test`.
- **Commits:** conventional lowercase prefix, reference the ticket, e.g. `feat: … (FE-06, #41)`. Every commit message body ends with:
  `Co-Authored-By: Claude Fable 5 <noreply@anthropic.com>`
- **Branch:** `feat/m1-first-run` (already created). Never commit to `main`.
